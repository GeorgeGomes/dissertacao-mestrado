Slide 7 — Algoritmo 1: Perceptron Estruturado com Relaxacao de Margem
Mostrar: Texto e pseudocodigo
Oq fazer: PSEUDOCODIGO COMPLETO do Perceptron Estruturado (Coelho, Borges
& Fonseca Neto, CILAMCE 2017, Secao 5.2):

  ENTRADA: X (m x d), y (m,), centroids (K x d)
           eta=0.001, C=1.0, delta_gamma=0.05, max_epochs=100, tol=1e-5
  SAIDA:   w (d,), gamma_otimo

  Inicializacao:
    w <- ones(d)
    alpha <- zeros(m)
    gamma_lo <- 0
    gamma_hi <- inf
    lambda <- 1 / C

  Busca binaria em gamma (loop externo, Eq. 31):
    repita ate |gamma_hi - gamma_lo| <= tol:
      gamma <- gamma_init (incrementado por delta_gamma a cada iter viavel)

      Perceptron interno para gamma fixo (loop sobre epocas):
        para cada ponto i em ordem aleatoria:
          k <- argmin distancia para centroide RIVAL
          l <- classe verdadeira (y[i])
          margem <- d_W(x_i, c_k) - d_W(x_i, c_l) + lambda * alpha[i]

          se margem < gamma * ||w||:                  (violacao, Eq. 28)
            g <- (x_i - c_l)^2 - (x_i - c_k)^2
            factor <- 1 - eta * gamma / ||w||
            w <- max(0, w * factor - eta * g)         (Eq. 29, projecao SDP)
            alpha <- alpha * factor; alpha[i] += eta

      se zero_violacoes:
        gamma_lo <- gamma; w_star <- w; gamma <- gamma + delta_gamma
      senao:
        gamma_hi <- gamma; gamma <- (gamma_lo + gamma_hi) / 2

  retorna w_star, gamma_lo

Hiperparametros (artigo, Secao 6, p. 16): eta=0.001, C in [0.1, 1.0]
(usamos 1.0).

Slide 8 — Algoritmo 2: NNLS (Lawson-Hanson 1974)
Mostrar: Texto e pseudocodigo
Oq fazer: PSEUDOCODIGO formal do NNLS (Mínimos Quadrados Não-Negativos)
para otimizacao inversa de metrica diagonal:

  ENTRADA: X (m x d), y (m,), centroids (K x d)
  SAIDA:   w (d,) com w >= 0

  Construcao da matriz A (m x d) e do vetor b (m,):
    para cada ponto i:
      l <- classe verdadeira (y[i])
      k <- centroide rival (k != l)
      para cada dimensao j:
        A[i, j] <- (x[i,j] - c[k,j])^2 - (x[i,j] - c[l,j])^2
      b[i] <- 1                  (margem-alvo; razao w invariante a escala)

  Resolver via scipy.optimize.nnls:
    w <- scipy.optimize.nnls(A, b)

  Formulacao matematica equivalente:
    min ||A w - b||^2   sujeito a  w >= 0

  Normalizacao opcional:
    w <- w / ||w||

  retorna w, gamma=0 (NNLS nao define margem geometrica explicita)

Algoritmo: metodo de conjunto ativo de Lawson-Hanson (1974).
Vantagens sobre Perceptron Estruturado:
  - Zero hiperparametros (deterministico, reproduzivel bit-a-bit)
  - Otimalidade global garantida (problema QP convexo)
  - Solucao unica em problemas bem-condicionados (rank(A) = d)
  - Tempo polinomial (sem busca binaria externa)
Cosseno tipico com W estimado pelo Perceptron: > 0.97 (validacao cruzada
do metodo).

Slide 24 — Nossa medida: consistencia interna, nao corretude
Mostrar: Texto
Oq fazer: Decisao metodologica explicita para a banca.

O trabalho mede CONSISTENCIA INTERNA do criterio decisorio do LLM:
  - FIDELIDADE = quanto a metrica W reproduz as decisoes do LLM (Problema A).
  - CONSISTENCIA = quanto W_A preve decisoes do LLM em B/C (Problemas B/C).
  - ACURACIA LLM-PERITO = quanto o LLM aprende com o perito (Problema E).

Decisao: NAO comparamos com ground truth nos problemas sinteticos.

Razao: H1 testa se o LLM tem criterio decisorio ESTAVEL, nao se ele
esta OBJETIVAMENTE CORRETO. Um LLM pode inverter classes e ainda assim
ser internamente consistente — saber se o criterio e "certo" e ortogonal
a pergunta de pesquisa.

Excecao: Bloco 3 (Problema G (peso x altura)) reportara acuracia vs rotulo real,
porque o orientador solicitou explicitamente no e-mail 22:04, ponto 4.
La ha ground truth significativo (homem/mulher) e classificador otimo
bayesiano (eliptico) conhecido — comparacao com a metrica aprendida
faz sentido pedagogico naquele contexto.
